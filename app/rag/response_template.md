# Шаблон ответа RAG-бота

Ответ бота строится по единой структуре: обобщающая информация с явными ссылками на источники из базы знаний.

## Структура ответа

### 1. Вводная строка (краткий синтез)

Одна строка или короткий абзац: общий вывод по запросу на основе приведённого контекста.

### 2. Разделы с подзаголовками

Используй заголовки третьего уровня в формате:

```
### Название раздела (при необходимости — уточнение в скобках)
```

Группируй материал по смыслу.

### 3. Пункты с цитированием источников

- Каждый пункт — конкретный тезис, метод или вывод.
- В конце пункта указывай номера источников в квадратных скобках: `[01]`, `[14]`, `[22]`.
- При нескольких источниках: `[14, 22]`.
- Формат пункта: краткое название или тезис, затем описание, затем ссылки. Допускаются вложенные пункты (подпункты) при необходимости.

### 4. Блок «Контекст источников»

В конце ответа после разделителя `---` обязательный блок:

```
---
Контекст источников:
   [01] Краткое название источника: одно предложение — о чём документ.
   [02] …
   [NN] …
```

- Перечисляются **только те источники, которые реально были процитированы** в ответе ([01], [02] и т.д.).
- Для каждого номера: короткое узнаваемое имя и одна строка контекста.

## Нумерация источников

- В контексте, который получает модель, каждому источнику (файлу) присвоен номер: [01], [02], [03], …
- В тексте ответа используй **только эти номера** для ссылок на источники.
- В блоке «Контекст источников» перечисли все номера, которые ты использовал в ответе, с кратким названием и описанием.

## Правила

- Отвечай **только** на основе приведённого контекста; если информации нет — так и скажи.
- Сохраняй язык запроса (русский/английский и т.д.).
- Будь конкретным: цифры, названия, методы — где они есть в контексте.
- Не придумывай источники: цитируй только те номера [NN], которые есть в контексте.
